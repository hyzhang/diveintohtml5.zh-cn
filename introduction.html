关于 HTML5 你需要知道的 5 件事

 

❧

1. It’s not one big thing

你可能会问：“如果旧的浏览器不支持 HTML5，我该怎么开始使用它呢？”但这个问题本身会让人产生误解。HTML5 不是一个大的、完整的东西，它是一个许多独立特性的集合。所以你没有办法检测“是否支持 HTML5”，因为那是没有意义的。但是你可以 检测是否支持各个特性，比如 canvas，video，geolocation。

你可能会认为 HTML 是很多标签和尖括号。那是它很重要的一部分，但不是全部。HTML5 规范还定义了这些元素怎样通过文档对象模型（DOM）和 JavaScript 进行交互。HTML5 不仅仅是定义了 <video> 标签，还有一套在 DOM 中操作视频对象的对应的 DOM API。你可以用这套 API 来检测不同视频格式的支持程度，对视频进行各种操作，并围绕 <video> 标签来创建丰富的用户体验。

第 2 章 和 附录 A 会教你怎样正确地检测 HTML5 特性。

2. You don’t need to throw anything away

无论你喜不喜欢，你都得承认 HTML 4 是有史以来最成功的标记格式。而 HTML5 在此基础上构建。你无需丢弃已有的标记，也不用重新学习已经知道的东西。如果昨天你的 Web 应用程序可以在 HTML 4 上运行，那么今天它也可以在 HTML5 上工作。就是这样。

现在，如果你想改进 你的 Web 应用程序，那你是来对地方了。这里有一个具体的例子：HTML5 不但支持所有 HTML 4 的表单控件，它也包含新的输入控件。有的是期待很久的，比如滑动条和日期选择器；而其他的一些控件则要更精细一些。比如，email 输入控件（商榷：此处 input type 翻译成了输入控件）看起来就和一个文本框一样，但移动浏览器会自定义它们的触屏键盘，使得输入邮件地址更容易。一些不支持 email 输入控件的旧的浏览器会把它看作一个普通的文本输入框，而表单也可以正常工作。这意味着就算你的访客还在冥顽不化地用着 IE 6，你也可以现在 就开始改进你的 Web 表单。

在 第 9 章 你可以看到 HTML5 表单所有的细节。（商榷：此处 gory 不知该怎么翻译）

3. It’s easy to get started

“升级”到 HTML5 可以像修改文档类型一样简单。文档类型应该总是在每个 HTML 页面的第一行。之前版本的 HTML 定义了许多文档类型，选择正确的那个可能还需要点技巧。在 HTML5 中，只有一种文档类型：

<!DOCTYPE html>

升级到 HTML5 不会影响现有的标记，因为 HTML 4 中定义的一些陈旧的元素在 HTML5 中仍然可以被渲染。但你仍然可以使用——并验证——新的语义元素，比如<article>，<section>，<header> 和 <footer>。在 第 3 章 你会学到这些新元素。

4. It already works

无论你是想在画布上绘图，播放视频，设计更好的表单，还是创建离线 Web 应用，你会发现它们在 HTML5 中都已经被很好地支持了。Firefox，Safari，Chrome，Opera 和移动浏览器都已经支持了画布（第 4 章），视频（第 5 章），地理位置（第 6 章），本地存储（第 7 章）还有许多其他的特性。Google 已经支持 microdata annotations（商榷：此处专有名词不知该如何翻译）（第 10 章）。甚至很少积极支持标准的微软，（商榷：此处翻译待商榷）也会在即将到来的 IE 9 中支持大多数 HTML5 特性。

本书的每一章都会包含大家都很熟悉的浏览器兼容性表。更重要的是，如果需要支持旧的浏览器，每章都对其他的选择作了直白（商榷：这里 frank 翻译不够准确）的讨论。一些 HTML5 特性，比如地理位置（第 6 章）和视频（第 5 章），最初由浏览器插件的形式提供。（Gears 和 Flash）（商榷：此处翻译需改进）其他的特性，比如画布（第 4 章），完全可以用 JavaScript 来模拟。本书会告诉你怎样以现代浏览器的原生特性为目标，并同时兼容旧的浏览器。（商榷：此处翻译待商榷）

5. It’s here to stay

Tim Berners-Lee invented the world wide web in the early 1990s. He later founded the W3C to act as a steward of web standards, which the organization has done for more than 15 years. Here is what the W3C had to say about the future of web standards, in July 2009:

Today the Director announces that when the XHTML 2 Working Group charter expires as scheduled at the end of 2009, the charter will not be renewed. By doing so, and by increasing resources in the HTML Working Group, W3C hopes to accelerate the progress of HTML5 and clarify W3C’s position regarding the future of HTML.

HTML5 is here to stay. Let’s dive in.
